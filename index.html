Architecture

The compiler paradigm shift.

Standard agent architectures put the LLM on the production hot path. We separate routing from compilation.

1. LLM as Compiler, Not Executor
We use local gemma4:12b models offline in batch to compile trace clusters into Elixir skills and generate synthetic test coverage. The LLM never touches hot-path production traffic for routine covered intents.
2. Provenance and Trust Contracts
The verifier stops responses that contain claims not matching specific database records. Every price, stock status, or shipment date is validated against catalog constraints.
3. Trace Moat Loop
Every session fallback is converted into a regression test. The system expands its coverage automatically as the offline compiler processes failure logs, creating a durable software asset library.

Strategy

Addressing the PMF framework.

Evaluating the business metrics through Sequoia Capital's PMF questions.

Why is this big?
$8B Commerce TAM
Existing helpdesks prove the workflow system of record. Hot-path agents prove autonomous resolution is possible. milli.run targets the middle path: agent-level coverage for repeated workflows, served through deterministic infrastructure where coverage is proven.
Why now?
LLM Commoditization
Inference is becoming increasingly standardized infrastructure. The differentiated value is captured in domain-specific workflows, evidence contracts, integration depth, and runtime reliability.
Why will it last?
Proprietary Test Moats
Every fallback trace can become a regression test case. The software library grows with usage when review, compilation, and deployment gates are enforced.
Why us?
Benchmark-First Science
We reject subjective demo scripts. The goal is to prove performance through reproducible fixtures, published scorer logic, raw outputs, and versioned benchmark runs.
Implementation

Pilot Installation Process

A low-risk, phase-by-phase onboarding roadmap to transition customer service operations to a compiled, deterministic runtime while maintaining safety gates and agent fallbacks.

Phase 1
Ingest & Train
We sync historical tickets, macros, policies, product catalog data, and order-state examples, establishing baseline coverage and training custom intent classifiers offline.
Phase 2
Compile & Verify
Core workflows are compiled into Elixir skills and integrated with your live APIs, running in shadow mode to audit accuracy against real agents.
Phase 3
Route & Scale
Routine tickets are routed gradually with automated verifier checks. Low-confidence interactions route to standard helpdesk queues.
W1–W2
Catalog Sync
Sync product catalog, order-state fixtures, policies, and active macros. Deploy shadow monitoring to baseline customer intents.
W3–W4
Extraction Training
Train FastText intent classifiers and spaCy extractors on historical tickets; establish confidence thresholds.
W5–W6
Skill Compiling
Compile core skills (WISMO tracking, return windows) into Elixir; connect live carrier and Shopify APIs.
W7–W8
Shadow A/B Run
Run milli.run in shadow mode. Verify product/variant match targets, evidence pass rates, escalation behavior, and latency against live agent sessions.
W9–W10
Controlled Launch
Route 5% of routine incoming tickets to milli.run. Enable automated verifier audits and escalation to the existing helpdesk queue.
W11–W12
Production Scale
Scale to 50% traffic. Stream shadow fallbacks to compiler to automatically expand intent skill library.

Autonomic support compiled to code. Not prompts.
