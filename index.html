The missing layer

The trillion-dollar prize only materializes if merchants can execute agent intent safely.

Consumer agents can express intent. Merchants still own the systems that make commerce real: catalog, price, inventory, promotions, loyalty, shipping, returns, refunds, fraud rules, and brand policy. The runtime must verify and constrain every action before it touches those systems.

Why chatbots failed

Conversation is not execution.

The first wave of shopping bots put probabilistic reasoning on the live customer path. That made commerce slower, more expensive, less predictable, and harder to certify. A customer may tolerate ambiguity during discovery. A merchant cannot tolerate ambiguity when an agent changes an order, approves a return, applies a discount, or commits to a delivery promise.

What changes now

Agents need a controlled merchant endpoint.

Agentic commerce turns shopping from search into delegation. That shifts the burden from answering questions to executing verified commercial intent. The merchant runtime becomes the trusted boundary between open-ended agents and operational systems of record.

System 2 · Offline

Learn, reason, compile.

LLMs and agents discover workflows, analyze failures, simulate edge cases, generate tests, and compile merchant-specific skills outside the production hot path.

System 1 · Online

Verify, route, execute.

Live interactions run through deterministic software: typed interfaces, evidence contracts, policy gates, regression tests, telemetry, rollback, and latency budgets.

What milli.run does

Turn commercial intent into verified merchant action.

milli.run compiles repeated commerce workflows into fast, observable execution paths. LLMs remain valuable for fallback and offline learning. Covered production paths run like software.

01

Understand intent

Classify customer or agent requests, extract structured slots, repair catalog-specific language, and identify when a workflow is covered or must escalate.

02

Verify evidence

Bind claims to merchant data before action. Prices, variants, order states, policies, stock, and delivery promises must map to records or approved rules.

03

Execute safely

Call typed tools against catalog, order, return, support, and fulfillment systems with policy gates, telemetry, confidence thresholds, and human fallback.

The operating lesson

Reasoning is not enough.

Ads systems infer intent indirectly. Chatbots talk around operational systems. Autonomous agents can act without merchant-grade controls. The next phase of commerce needs a runtime that gives merchants a trusted endpoint for intent: evidence-bound, policy-constrained, observable, and fast enough for production.

Deployment path

Start with support workflows. Expand into agentic commerce execution.

Support is the natural wedge because it already contains repeated commercial intent: where is my order, can I return this, which variant fits, can I exchange it, is this eligible, what can you recommend.

Phase 1

Ingest

Sync tickets, macros, policies, product catalog, order states, returns, and fulfillment data. Establish baseline coverage and evidence requirements.

Phase 2

Compile

Train routers and extractors, compile covered workflows, generate regression tests, and run in shadow mode against real support sessions.

Phase 3

Route

Route covered traffic through deterministic execution paths. Escalate low-confidence interactions and feed fallbacks back into the compiler loop.

Architecture thesis

LLMs learn offline. Deterministic merchant software executes online.

milli.run turns agentic commerce from runtime reasoning into compiled execution: fast enough for customer support, reliable enough for merchant operations, and controlled enough for brands and retailers to trust.
